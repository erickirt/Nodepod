nodepod · SharedArrayBuffer opt-out

By default nodepod uses SharedArrayBuffer for sync fs reads, execSync/spawnSync, and threaded wasi modules (rolldown, lightningcss, tailwind-oxide). Pass enableSharedArrayBuffer: false to force it off. Dependent features either degrade to async message passing, or lazy-throw when actually invoked.

1 · Default boot (SAB on)

Happy path, matches what every other example does. Both a regular script and execSync work.

2 · Opt-out: { enableSharedArrayBuffer: false }

Boot still succeeds, you get a warn in devtools. isSharedArrayBufferEnabled reads false. Regular scripts and async fs still work. Common builtins like echo, ls, cat, pwd, version checks etc. have fast-paths that run in pure JS and still work. Anything that needs a real subprocess (like execSync("node -e ...")) throws with a message that tells you how to turn SAB back on.
